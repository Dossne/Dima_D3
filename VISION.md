# VISION.md — Tap Miner: Collapse Run
*Единый источник правды: что мы делаем, зачем и как это ощущается*

---

## Одна фраза

> Игрок удерживает майнера в шахте, свайпая влево/вправо между тремя лейнами. Снизу непрерывно поднимаются блоки — если не дробить их вовремя, они выталкивают игрока к потолку. Каждый ран — 20–50 секунд. Смерть → апгрейд → снова.

---

## Референсы

| Игра | Что берём |
|------|-----------|
| **Tomb of the Mask** | Вертикальный скролл, portrait, читаемые паттерны, аркадный темп, мгновенный рестарт |
| **Archero** | Upgrade-петля между ранами, idle-lite прогрессия, ощущение роста |
| **Downwell** | Нарастающая угроза, roguelite ощущение, короткие раны |
| **Crossy Road / Flappy Bird** | Простое управление, честная смерть, моментальный рестарт |

**Формула:** `Tomb of the Mask readability + mining fantasy + Archero-style meta-lite`

## Что видит игрок после старта

```
┌─────────────────────────┐
│ ━━━━━━━━━━━━━━━━━━━━━━━ │  ← ПОТОЛОК (фиксирован, не двигается)
│              [42.5m]    │  ← HUD: глубина
│                         │
│                         │
│        [🧑‍⚒️]             │  ← персонаж: baseY ~35% снизу, может подниматься
│  Lane0  Lane1  Lane2    │
│                         │
│  [ B ]  [   ]  [ B ]   │
│  [   ]  [ R ]  [   ]   │  ← блоки / монеты / бомбы поднимаются снизу
│  [   ]  [   ]  [ S ]   │
│  [ S ]  [ S ]  [ S ]   │
│                         │
│       [COINS: 42]       │
└─────────────────────────┘
```

- Блоки поднимаются **снизу вверх** непрерывно
- **Потолок фиксирован** — не двигается
- Игрок находится на **baseY (~35% снизу)**. Если блок в его лейне — его выталкивает вверх
- Дробишь блок вовремя → падаешь обратно к baseY
- Не дробишь → тебя прижимает к потолку → смерть
- Первые **5–8 секунд** — максимально простые сегменты, туториал не нужен

---

## Лейны и движение персонажа

**3 фиксированных лейна** (Left / Center / Right). Лейны не меняют положение горизонтально.

```
Lane 0     Lane 1     Lane 2
(Left)    (Center)   (Right)
  X=-w      X=0        X=+w
```

**Управление:**
- Swipe left → переход в соседний левый лейн
- Swipe right → переход в соседний правый лейн
- Tap → дробит блок в текущем лейне (если блок есть)
- Движение **дискретное** — всегда на якорь лейна
- Время перехода: **120 мс**
- В буфер принимается **1 следующий input** во время перехода
- Auto-correction **запрещён** — куда свайпнул, туда и пошёл

## Что происходит на экране во время рана

### 1. Блоки и сегменты шахты

Поле разбито на сегменты. Каждый ряд — 3 ячейки. Содержимое:

| Символ | Что это |
|--------|---------|
| **B** | Block — дробится тапом. Пока в лейне игрока — выталкивает его вверх |
| **R** | Reward — монета/кристалл, собирается автоматически при прохождении через лейн |
| **H** | Hazard — бомба/ловушка. Не блокирует, но наносит HP урон и замедляет скролл на 2 сек |
| **S** | Safe — пустая ячейка, без эффектов |

### 2. Механика выталкивания

- Мир **скроллится непрерывно** с постоянной скоростью
- Когда блок достигает лейна игрока → игрок начинает подниматься вместе с блоком
- **Tap** в нужный момент → блок дробится → игрок возвращается к baseY
- Несколько блоков подряд без свободных лейнов → быстрый рост давления

### 3. Потолок

- **Потолок** — фиксированная граница в верхней части экрана
- Игрок **видит своё положение напрямую** — он физически поднимается к потолку. Отдельная полоска давления не нужна
- При приближении к потолку — ambient-сигнал: vignette / screen shake (детали в отдельной задаче по juice)
- Игрок касается потолка → **мгновенная смерть** (cause = PRESSURE)

### 4. Hazard'ы

Не блокируют движение. При контакте:
- Снимают HP
- Замедляют скорость скролла на 2 секунды

Всегда **телеграфируются** (цвет, анимация) — смерть от hazard'а должна быть объяснима.

Типы (MVP): взрывные блоки, шипы, ядовитый газ.

### 5. Лут и монеты

Собираются **автоматически** при прохождении через лейн с Reward. CoinLabel обновляется мгновенно. Кристаллы (редкий лут) видны заранее — провоцируют риск смены лейна.

### 6. HUD (всегда на экране)

- **DepthLabel** — текущая глубина, обновляется каждый кадр
- **CoinLabel** — монеты текущего рана
- **HpBar** — текущее здоровье

## Паттерны сегментов — примеры

### Early (Depth 0–10) — читаемость важнее давления

```
Lane0  Lane1  Lane2
  B      R      S    ← блок слева, монета в центре, справа — свободно
  S      S      S
  S      H      S    ← hazard в центре, telegraph виден
  S      S      S
```

Всегда есть очевидный выход. Учим: блок = надо тапать, R = монета даром.

### Mid (Depth 11–25) — давление нарастает

```
Lane0  Lane1  Lane2
  B      H      B    ← два блока, центр — hazard: нужно выбрать лейн
  S      S      S
  R      B      S    ← монета в опасном лейне
  S      S      S
```

Нельзя просто стоять — блоки выталкивают быстро. Нужно принимать решения.

### Late (Depth 25+) — skill check

```
Lane0  Lane1  Lane2
  B      B      S    ← только lane2 безопасен
  H      S      B
  S      B      H
  B      S      S    ← нужно держать lane1 → lane2 → lane1
```

Тесные маршруты, ошибка = быстрый потолок.

### Запрещённые паттерны

- `B B B` в одном ряду — невозможно уклониться, нарушает Fairness Contract
- Block без доступного лейна рядом — unfair
- Hazard без telegraph — смерть необъяснима
- 3+ ряда блоков подряд без зазора — читаемость рушится

## Как меняется сложность и визуал по глубине

| Глубина | Геймплей | Цвет среды |
|---------|----------|------------|
| **0–10 Early** | Медленный скролл, простые паттерны, очевидный выход | Серый камень `#48484A`, тёплые коричневые акценты |
| **11–25 Mid** | Скролл быстрее, плотные паттерны, hazard'ы нарастают | Тёмный камень `#2C2C2E` + синие кристаллы `#00C7FF` |
| **25+ Late** | Быстрые решения, тесные маршруты, высокое давление | Почти чёрный `#1C1C1E` + purple neon glow `#BF5AF2` |

## Петля смерти и рестарта

```
Ран (20–50 сек)
      ↓
    Смерть
      ↓
 Results Screen
 (глубина + монеты + breakdown)
      ↓
 Upgrade Screen
 (купить 1–2 апгрейда)
      ↓
 Новый ран — быстро
```

**Главный принцип:** от смерти до следующего тапа "старт" — **не более 5–10 секунд**.

## Pillars — чем руководствуемся при каждом решении

| Пиллар | Что это значит на практике |
|--------|---------------------------|
| **Мгновенно понятно** | Управление и цель ясны за 5 секунд без туториала |
| **Плотный темп** | Игрок почти всегда принимает микро-решение |
| **Риск против награды** | Глубже = быстрее = опаснее = выгоднее. Жадность — движущая сила |
| **Быстрый рестарт** | Смерть не штрафует, а перезапускает |
| **Лёгкая мета** | Прогресс ощущается уже после 2–3 забегов |

---

## Fairness Contract — что никогда не нарушаем

- **Readable route** — всегда есть хотя бы один лейн без блока
- **Fair death** — тестер должен объяснить причину смерти без помощи разработчика
- **Telegraph** — hazard виден до контакта
- **Pressure** — потолок наказывает за бездействие, не за нормальную игру
- **Restart** — смерть быстро переходит в следующую попытку
